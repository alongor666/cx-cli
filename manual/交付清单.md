## 🎯 使用流程

### 第一次使用（推荐自动化）

1. **先在终端 cx login**（1 分钟；masked PAT 输入，不写进任何脚本 — 安全约定 PR #669）
   - 打开 PowerShell：`cx login`
   - 验证：`cx whoami` 不报 401

2. **配置脚本**（1 分钟）
   - 打开 `capture-screenshots-v2.ps1`
   - 仅修改 `$QUERY_YEAR = "有数据的年份"`（不要碰 PAT 字段，已是占位符）

3. **运行脚本**（5-10 分钟）
   - 双击 `开始截图.bat`
   - 或右键 `capture-screenshots-v2.ps1` → 使用 PowerShell 运行
   - 按提示按 10 次回车键

3. **补充架构图**（5 分钟）
   - 用 Figma/Draw.io 绘制系统架构图
   - 保存为 `images/placeholder-architecture.png`

4. **验证效果**（2 分钟）
   - 浏览器打开 `index.html`
   - 按 F5 刷新
   - 按 F11 全屏演示

### 手动截图（备选）

如果自动化脚本不适用，按照 `截图指南.md` 手动截图。

## 🆘 故障排除

### 脚本无法运行
- **错误**：执行策略错误
- **解决**：运行 `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass`

### cx 命令不可用
- **错误**：`'cx' 不是内部或外部命令`
- **解决**：
  1. 检查 `C:\tools\cx\` 目录
  2. 检查系统 PATH 设置
  3. **重启命令行窗口**
  4. 运行 `cx --version` 测试

### 登录失败
- **错误**：`Authentication failed`
- **解决**：
  1. 检查 PAT 格式（`cx_pat_<id>.<secret>`）
  2. 确认 token 未过期
  3. 检查网络连接

### 图片不显示
- **错误**：浏览器显示占位符文字
- **解决**：
  1. 检查文件名是否匹配（区分大小写）
  2. 按 `Ctrl+F5` 强制刷新
  3. 检查浏览器控制台（F12）是否有 404 错误
