# 自动化截图脚本使用说明

## 概述

提供了两个 PowerShell 脚本，可自动执行 cx-cli 命令并截图：

| 脚本 | 说明 | 推荐度 |
|------|------|--------|
| `capture-screenshots-v2.ps1` | 交互式，按步确认 | ⭐⭐⭐⭐⭐ |
| `capture-screenshots.ps1` | 自动化，无需确认 | ⭐⭐⭐ |

---

## 前置条件

### 1. 系统要求
- ✅ Windows 10/11
- ✅ PowerShell 5.1 或更高（默认已安装）
- ✅ 已安装 cx-windows.exe 并添加到 PATH

### 2. 检查 cx 是否可用
打开 PowerShell，输入：
```powershell
cx --version
```

如果显示版本号（如 `0.1.0`），说明已正确安装。

### 3. 准备 PAT Token
联系管理员获取 PAT token，格式：`cx_pat_<id>.<secret>`。**不要**把真 PAT 写进任何脚本文件 — 走交互式 `cx login` 输入（masked）。

---

## 方法一：交互式脚本（推荐）⭐

### 使用步骤

1. **先终端外手动 cx login**（masked PAT 输入，不入任何文件 — 安全约定 PR #669）

   ```cmd
   cx login
   cx whoami  # 验证不报 401
   ```

2. **编辑脚本配置**（仅修改 QUERY_YEAR，不要碰 PAT 字段）

   打开 `capture-screenshots-v2.ps1`，修改 `$QUERY_YEAR`：
   ```powershell
   $QUERY_YEAR = "2026"           # 根据实际数据修改
   ```

3. **运行脚本**

   **方式 A**：右键点击脚本文件 → 选择"使用 PowerShell 运行"

   **方式 B**：在 PowerShell 中运行：
   ```powershell
   cd "C:\path\to\cx-cli\manual"
   .\capture-screenshots-v2.ps1
   ```

4. **按提示操作**

   脚本会：
   - 检查 cx 命令是否可用 + 是否已 login（cx whoami 不报 401）
   - 清理旧截图（询问确认）
   - 逐个执行命令并提示截图（login 步骤走 DisplayOnly，不真跑）
   - 每步按回车键继续

5. **查看结果**

   脚本完成后，所有截图保存在 `images/` 目录（已被 cli/.gitignore + workflow rsync 双层排除，不会同步到公开 cx-cli）

### 优点
- ✅ 每步都有提示，不容易出错
- ✅ 可以调整窗口大小和位置
- ✅ 可以看到命令执行结果
- ✅ 有详细的完成报告

### 缺点
- ❌ 需要手动按 10 次回车
- ❌ 耗时约 5-10 分钟

## 方法二：全自动脚本

### 使用步骤

1. **先终端外手动 cx login**（同方法一第 1 步，不要把真 PAT 写进脚本）

2. **编辑脚本配置**（仅修改 QUERY_YEAR）

   打开 `capture-screenshots.ps1`，修改 `$QUERY_YEAR`：
   ```powershell
   $QUERY_YEAR = "2026"           # 根据实际数据修改
   ```

3. **运行脚本**

   ```powershell
   cd "C:\path\to\cx-cli\manual"
   .\capture-screenshots.ps1
   ```

4. **等待完成**

   脚本会自动执行所有命令并截图，无需干预

### 优点
- ✅ 完全自动化
- ✅ 快速（约 3-5 分钟）

### 缺点
- ❌ 无法中途调整窗口
- ❌ 如果出错需要全部重来

## 截图效果优化建议

### 1. PowerShell 主题设置

**推荐使用深色主题**（更符合 PPT 风格）：

```powershell
# 设置 PowerShell 控制台颜色
Set-PSReadlineOption -Colors @{
    Command = 'Green'
    Parameter = 'Cyan'
    String = 'Yellow'
    Comment = 'Gray'
    Keyword = 'Magenta'
}
```

**或使用 Windows Terminal**（推荐）：
1. 安装 Windows Terminal（Microsoft Store）
2. 打开设置 → 配置 PowerShell 配置文件
3. 配色方案选择"Campbell"或"One Half Dark"

### 2. 窗口大小建议

```powershell
# 设置窗口大小（推荐）
$Host.UI.RawUI.WindowSize = New-Object System.Management.Automation.Host.Size(140, 30)
```

或在 Windows Terminal 设置中：
- 宽度：1400-1600 px
- 高度：900-1000 px
- 字体：Cascadia Code 或 Consolas，14-16pt

### 3. 字体大小调整

如果截图后字体太小：

```powershell
# 放大控制台字体
$Host.UI.RawUI.BufferSize = New-Object System.Management.Automation.Host.Size(140, 3000)
```

## 常见问题

### Q1: 脚本无法运行（执行策略错误）

**错误信息**：
```
无法加载文件 capture-screenshots-v2.ps1，因为在此系统上禁止运行脚本
```

**解决方案**：
```powershell
# 临时允许脚本执行（当前会话）
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

# 然后重新运行脚本
.\capture-screenshots-v2.ps1
```

### Q2: cx 命令不可用

**错误信息**：
```
✗ 错误: cx 命令不可用
```

**解决方案**：
1. 检查 `C:\tools\cx\` 目录是否存在 `cx-windows.exe`
2. 检查系统 PATH 是否包含 `C:\tools\cx\`
3. **重启所有 PowerShell/CMD 窗口**
4. 运行 `cx --version` 测试

### Q3: 登录失败

**错误信息**：
```
✗ Authentication failed: Invalid PAT
```

**解决方案**：
1. 检查 PAT token 格式是否正确（`cx_pat_<id>.<secret>`）
2. 确认 token 未过期
3. 检查网络连接
4. 确认后端地址是否正确（内网环境需要配置）

### Q4: 查询无数据

**现象**：`cx query KPI --year=2026` 返回空结果

**解决方案**：
1. 修改脚本中的 `$QUERY_YEAR` 为有数据的年份
2. 或尝试其他查询参数
3. 检查 PAT 对应的数据权限范围

### Q5: 截图模糊

**原因**：窗口太小或分辨率不足

**解决方案**：
1. 调整 PowerShell 窗口大小（至少 1400×900）
2. 使用 Windows Terminal（渲染更清晰）
3. 调整 DPI 设置（100% 或 125%）
